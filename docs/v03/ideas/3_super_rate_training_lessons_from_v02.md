# NERS590 v02 SUPER RATE Training Review

## Scope

This report reviews the first dedicated training experiments for predicting `SUPER RATE (CC/S)` on the active `NERS590_data_v02` dataset. These experiments follow the SUPER RATE-specific cleaning and modeling plan in which the target is treated as a separate sparse prediction problem rather than as a simple relabeling of the existing `RATE CONST` task.

The training workflow used the same merged multi-file dataset and the same power-aware input space as the current v2 pipeline, but changed the target construction and evaluation rules to match the much sparser `SUPER RATE` output. Citations: [ners590_v02_super_rate_experiment_plan.md](/Users/bingqingwang/Desktop/UMich/590_Machine%20learning/project/docs/v03/ideas/1_super_rate_prediction_plan_inspiration.md), [ners590_v02_super_rate_analysis_report.md](/Users/bingqingwang/Desktop/UMich/590_Machine%20learning/project/docs/archive/active_v02/ners590_v02_super_rate_analysis_report.md), [experiment_comparison_summary.csv](/Users/bingqingwang/Desktop/UMich/590_Machine%20learning/project/results/ners590_v02_super_rate/experiment_comparison_summary.csv).

## What Was Trained

The experiments used one unified power-aware model per experiment set, trained on the merged dataset rather than training separate models for each power file. Inputs were the species composition features plus `log10(E/N)` and `log10(power_mj)`. For `SUPER RATE`, the pipeline first derived the full 204-output target table from the parsed long reaction table, then removed the always-zero channels from the modeled target matrix.

That cleaning step was substantial: out of the full `204` SUPER RATE outputs, only `94` were kept for direct modeling and `110` were dropped as constant-zero channels. Final predictions were then reconstructed back to the full 204 outputs by reinserting zeros for those dropped channels. This matters when reading the metrics: the “active target” summaries describe the learned nontrivial subset, while the “full reconstructed” summaries describe the final 204-output prediction vector. ## The Two Experiment Sets

Two experiment sets were run.

The first was the random-case experiment in [training_random_case](/Users/bingqingwang/Desktop/UMich/590_Machine%20learning/project/results/ners590_v02_super_rate/training_random_case). Its purpose was interpolation performance: train and test on a random mixture of cases drawn from all powers, density groups, and local cases. This is the easiest and most general “can the model fit the observed dataset well?” benchmark.

The second was the unseen-power experiment in [training_power_holdout_5mJ](/Users/bingqingwang/Desktop/UMich/590_Machine%20learning/project/results/ners590_v02_super_rate/training_power_holdout_5mJ). Its purpose was power generalization: train on `1mJ` to `4mJ`, then test on a fully held-out `5mJ` file. This is scientifically more demanding because it asks whether the unified model can transfer to an unseen power level rather than only interpolate inside the observed mixed-power cloud.

Both experiments evaluated the same curated baseline family:

- direct Ridge
- direct ExtraTrees
- sparse-aware two-stage ExtraTrees
- direct MLP
- latent-output ExtraTrees

## Which Model Won

The same model won both experiment sets: `direct__extra_trees__all_inputs_plus_log_en_log_power__n_200__d_12__leaf_1`. This is an important result because it says the best baseline was not the sparse-aware two-stage model and not the latent PCA model. In other words, for the current v2 dataset size and structure, a direct tree ensemble on the full power-aware input set already captures the `SUPER RATE` mapping extremely well.

The leaderboard also shows that the two-stage idea was not useless, but it was not the strongest option. On the composition-PCA feature path, the two-stage ExtraTrees model was close to the direct ExtraTrees model, especially in the power-holdout setting. But once all inputs were exposed directly, the plain direct ExtraTrees model still won by a clear margin. The MLP baseline was competitive in absolute terms but consistently weaker than the tree baselines, and the latent-output ExtraTrees model was clearly worse than direct prediction. That suggests target compression is not helping this problem at the current scale, and explicit sparse gating is not yet buying enough to justify its added complexity. ## Main Results

The random-case run was extremely accurate. On the full reconstructed 204-output vector, it achieved log-RMSE `0.001243`, log-MAE `0.000057`, and log-R2 `0.999999`. On the retained nontrivial targets only, it achieved log-RMSE `0.001831` and log-R2 `0.999998`. The median absolute relative error on positive targets was about `4.90e-05`, and about `99.98%` of positive predictions were within 10% relative error.

The unseen-`5mJ` power-holdout run was slightly harder, but still extremely strong. On the full reconstructed 204-output vector, it achieved log-RMSE `0.003482`, log-MAE `0.000245`, and log-R2 `0.999973`. On the retained nontrivial targets only, it achieved log-RMSE `0.005130` and log-R2 `0.999941`. The median absolute relative error was about `3.29e-05`, and about `99.80%` of positive predictions were still within 10% relative error.

So the overall conclusion is that the new v2 dataset makes `SUPER RATE` much more learnable than one might expect from its sparsity alone. Even the harder unseen-power test remains highly accurate. ## Why the Full Reconstructed Metrics Look Even Better

The gap between the “active target” metrics and the “full reconstructed” metrics is expected and should not be misread. The full 204-output evaluation includes the `110` channels that were dropped as constant-zero targets and then reconstructed exactly as zeros. Those channels are truly trivial for the dataset, so they improve the overall full-vector metrics without meaning the learned part of the problem got easier.

That is why the retained-target metrics are important. They tell us how well the model predicts the `94` nontrivial `SUPER RATE` channels that actually contain information. The fact that those active-target metrics are still extremely strong is what makes the result genuinely convincing. ## Where the Model Is Still Hardest

The hardest reactions are not random. In both experiment sets, the largest log-space errors cluster around the oxygen vibrational channels and nearby low-magnitude structured processes: `H2 > H2(V=2)`, `O2-VIB 3`, `O2-VIB 2`, `H2 > H2(V=1)`, `O2-VIB 4`, `O2-VIB 1`, and `O-1D` appear repeatedly among the worst reactions. These are also exactly the reactions that earlier analysis flagged as relatively power-sensitive, so the result is physically consistent.

The hardest cases are also structured. When test errors are aggregated by local case, the largest mean log-RMSE values occur at local cases `3-4` in both experiments, with a milder but still elevated difficulty at local cases `1-2` and `5-6`. Error drops sharply after local case `7`. So the model’s remaining challenge is concentrated in the early part of the 29-point sweep rather than in the high-activity late sweep. That matches the earlier SUPER RATE analysis, which showed that the target changes regime early in the sweep as activation ramps up. ## Error Behavior by Magnitude

The relative-error summaries show a healthy pattern: for medium and large `SUPER RATE` values, prediction error is extremely small. The hardest region is the very tiny positive tail, especially around `1e-18` to `1e-16`, where relative errors inflate because the denominator is tiny. This is not a sign of a generally bad model; it is the expected behavior for ratio-based metrics on near-zero targets.

For the more practically important ranges, performance is excellent. In both experiments, once the true values are above roughly `1e-12`, the median absolute relative error drops to the `1e-4` to `1e-5` range. That is why the global median relative errors remain so small even though the smallest bins are much harder. ## Interpretation and Next Recommendation

The first-phase conclusion is strong: `SUPER RATE` is not only predictable, it is predictable at very high precision with a relatively simple direct ExtraTrees model once we clean the targets properly. For the current dataset, that means the best next modeling move is not necessarily to add complexity immediately. The current direct tree baseline is already an excellent benchmark.

The two most useful next steps would be:

1. keep this direct ExtraTrees model as the main production baseline for `SUPER RATE`
2. only then test a shared-trunk multi-task model with separate heads for `RATE CONST` and `SUPER RATE`, using this single-task SUPER RATE baseline as the reference

At this point, a joint model is worth exploring because many SUPER RATE channels are structurally related to RATE CONST, but it should be judged against this new baseline rather than replacing it blindly.
